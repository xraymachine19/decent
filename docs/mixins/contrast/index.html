.contrast()

Finds a contrastful color. Outputs it to the given property as a rule, usually color (the default), border-color or background-color.

Arguments

  • @property: The name of a CSS property
  • @background-color: A color to contrast against
  • @standard: The contrast ratio standard to meet
  • @mode: Which contrast method to use
    • tint: Add white to find contrast
    • shade: Add black to find contrast
    • auto: (default) Add whichever has more contrast potential
mixin · outputs a CSS rule

Examples

Basic usage

By default, contrast sets the color property to a contrastful value against @content-background.

p {
    .contrast();                        
}
<p>A paragraph with default text contrast.</p>

Output:

A paragraph with default text contrast.

Different background colors

Sometimes background colors will be an awkward color, like this CSS color blue, where our text color doesn't work as well:

<p style="background:blue;">This is text on blue</p>

This is text on blue

We can use .contrast() to make the text meet our project's contrast standards with this .text-on-blue class :

p.text-on-blue {
    .contrast(color, blue);                        
}
<p style="background:blue;" class="text-on-blue">This is text on blue</p>

This is text on blue

Now the text is more readable and meets the project's standards.

Of course, the official Decent way of making something blue is to simply add the blue class. Instead of CSS's take on blue, it uses your project's take on blue.

This is a blue paragraph with text-padding too.

To set both the text color and background color, use .colorize().

Using different contrast standards

All contrasting is done according to a standard, a contrast ratio which is usually 4.5 or 7 for text, according to WCAG standards.

WCAG standards are lower for large text, like headlines. In your project, you may wish to set headline text contrast to a lower standard (WCAG suggests at least 3) so that your headlines are more colorful:

.headline-elements({
    .contrast(color, blue, 3); 
});

This targets all headlines using the headline ruleset mixin, applyin the same contrast algorithm as before but stopping when a contrast ratio of 3 is met. Not how the headline here is less contrastful than the body text from before:

A softer-contrasted headline

This is text on blue

Outputting different properties

Suppose we want to give a border to the paragraph in this new blue-backgrounded element.

To do this, we give .contrast() the border-color property, along with a higher contrast standard to stand out more:

p {
    .contrast(border-color, blue, 7); 
}

This gives us:

A softer-contrasted headline

This is text on blue

Forcing a contrast mode

We may find that the automatic contrasting doesn't pick the color we had in mind. If we want to force the border-color in the prior example to use black to find contrast, we can set the @mode argument to shade.

p {
    .contrast(border-color, blue, 7, shade); 
}

A softer-contrasted headline

This is text on blue

Now the border around the paragraph will always tend to be darker.

It's important to keep in mind that this may reduce contrast possibilities. Forcing a very light color to use the tint mode, for example, will never produce contrastful text.

For more on this concept, read about .set-contrast-floor() and .set-contrast-ceiling(), which find how much room there is available for either tinting or shading operations.
