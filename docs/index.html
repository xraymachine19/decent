From a to z-index, Decent supports all HTML elements and all CSS properties.

You might not have known about the kbd HTML element, but your Decent project does.

It's even got a dedicated kbd.less file:

kbd {
    .default(border-radius);
    .colorize(@black, 9);
    padding: @text-padding;
}

In Decent, every HTML element has its own file, meaning all of HTML is fully supported right out of the box. Each element is handily organized into its own file, quickly accessible with Ctrl + P (That's the kbd element, by the way).

CSS properties are fully supported by way of Less variables in properties.less and the .default() mix-in, seen above in kbd.

The .default() mix-in takes any CSS property and outputs it as a CSS custom variable, meaning everyone on your project can use the framework-like features of Decent, even if they write only CSS and not Less.

border-radius: var(--border-radius);

The :root element will contain your project's reference to a default border-radius value.

To reuse "border-radius" yet again, your project will also have out-of-the-box support for using any CSS property as a class name, again applying the default value.

This makes it easy for other sites— or other parts of your own site— to use your prescribed defaults and values.

Setup and testing

  1. Run python -m SimpleHTTPServer in the decent directory.
  2. Follow the directions there (or just go to localhost:8000)
  3. Edit index.html or the less, then refresh to test stuff.
  4. index.html isn’t the public splash page for github but you can put that kinda content there if you’d like. It’s really just a scratchpad. Test-style setups (gray-scale contrasting table for example) are good here too.

Compiling the Less

  1. Run ‘lessc index.less css/index.css’.

Core ideas

Accessibility

Decent aims to preserve and emphasize accessibility.

Color contrast

  • Mixins ensure developers meet WCAG standards
  • Mixins ensure design consistency

Lean performance

  • Decent allows for a leaner DOM, which makes everything (CSS, JS) faster
  • Decent itself compiles quickly, picking the right colors out of millions in seconds

Environment

Decent is a fuller coding environment for Less.

HTML & CSS concepts

  • All HTML elements have a file
  • All CSS properties have a variable

Richer mixins

  • Each mixin has its own file
  • Mixins use import statements to list dependencies

Design fundamentals

Decent gives designers fundamental tools for building UIs.

Patterns

Semantic colors

Decent uses very few classes. Semantic classes, one of the few exceptions, come with an inherent color association.

Palette colors

  • action: A version of the base color
  • reaction: A complement to action

Informational colors

  • info: An info-conveying highlight
  • warning: An urgent info-conveying orange

Transactional colors

  • success: A reassuring green
  • failure: An alarming red

Unique colors

Every color in a Decent palette also doubles as a CSS class and/or a Less variable. The palette is based on a base color:

base-color

Here, it's given to you. But in your own project you can define it to be whatever you want.

Semantic colors

Semantic colors are designed to be one-word and self-explanatory. Each has a corresponding Less variable (or CSS custom property).
